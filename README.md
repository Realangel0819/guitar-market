# 🎸 Guitar Market – 기타 중고거래 플랫폼 (Backend Project)

## 1. 프로젝트 소개

**Guitar Market**는 기타 및 악기 장비를 중심으로 한 중고거래 플랫폼입니다.
기존 중고거래 서비스에서 자주 발생하는 **카테고리 확장 문제, 거래 상태 관리, 이력 추적의 어려움**을 해결하는 것을 목표로 설계·구현한 백엔드 중심 프로젝트입니다.

본 프로젝트는 단순한 CRUD 구현을 넘어서,

* 실제 서비스에서 발생하는 거래 흐름
* 상태 변경에 따른 데이터 정합성
* 확장 가능한 도메인 모델링

을 고려하여 설계되었습니다.

---

## 2. 기획 배경

기타 및 악기 중고거래는 일반 중고거래와 달리

* 악기 종류가 매우 세분화되어 있고
* 장비 특성상 카테고리 확장이 빈번하며
* 거래 과정에서 상태 변경이 자주 발생합니다.

기존 커뮤니티 기반 중고장터(Mule 등)는 기능적으로는 충분하지만,
**데이터 구조 관점에서 확장성과 명확한 거래 흐름을 설명하기 어렵다**는 한계가 있다고 판단했습니다.

이에 따라 본 프로젝트에서는

> “실제 서비스라면 어떻게 설계해야 할까?”
> 라는 질문을 중심으로 도메인 설계를 진행했습니다.

---

## 3. 핵심 기능

### 3.1 회원 기능

* 회원 가입 / 로그인
* 본인 인증 여부 관리
* 판매자/구매자 구분 없는 단일 User 모델

---

### 3.2 카테고리 시스템

#### 대분류

* 일렉기타
* 어쿠스틱 / 클래식 기타
* 베이스
* 앰프
* 이펙터
* PA / 키보드
* 기타 장비

#### 세부분류 (예시)

* 기타 → 픽업 / 부품
* 앰프 → 진공관 / 솔리드 / 모델링
* 이펙터 → 싱글 이펙터 / 멀티 이펙터

➡️ **자기참조(Category Self Join) 구조**로 설계하여
새로운 악기 종류가 추가되더라도 DB 구조 변경 없이 확장 가능하도록 설계했습니다.

---

### 3.3 판매글(상품) 기능

* 판매글 작성 / 수정 / 삭제
* 판매 상태 관리

  * 판매중
  * 예약중
  * 판매완료
* 판매 가격 설정
* 다중 이미지 업로드
* 조회수 집계
* 게시글 번호 자동 생성
* 공지글 등록 가능

---

### 3.4 거래 시스템

* 구매 요청
* 판매자 수락
* 거래 완료
* 거래 상태 변경 이력 관리

## 4. ERD 설계

> 본 프로젝트의 핵심은 **ERD 설계**입니다.

### 4.1 설계 원칙

1. **확장성**

   * 카테고리 및 상품 유형 증가에 유연하게 대응

2. **상태 분리**

   * 판매글 상태(Post Status)와 거래 상태(Transaction Status)를 분리

3. **이력 관리**

   * 거래 상태 변경 이력을 별도 테이블로 관리

---

### 4.2 주요 테이블 설명

#### User

* 회원 정보 관리
* 판매자/구매자 구분 없이 단일 테이블 사용

#### Category

* 자기참조 구조를 통한 무한 Depth 카테고리 지원
* 대분류 / 세부분류 유연하게 구성 가능

#### Product(Post)

* 판매글 단위의 상품 정보
* 판매 상태(판매중/예약/완료) 관리
* 거래와 독립적인 생명주기 보유

#### Transaction

* 실제 거래 단위
* 구매자 / 판매자 명확히 분리
* 거래 상태 관리

#### TransactionHistory

* 거래 상태 변경 이력 저장
* 추후 분쟁 처리, 감사 로그로 확장 가능
